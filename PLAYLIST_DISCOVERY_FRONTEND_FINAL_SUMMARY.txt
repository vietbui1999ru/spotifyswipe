FEATURE FLOW
================================================================================

User Journey:
  1. Click "Browse Playlists" in navigation
  2. Presented with 12 genre/mood options
  3. Clicks preferred genre (e.g., "Pop")
  4. Playlist search results displayed
  5. Clicks a playlist
  6. Navigated to swipe page with playlist tracks
  7. Swipes through playlist tracks
  8. Saves liked tracks to playlist

Technical Flow:
  navigate to /dashboard/discover
    ‚Üì
  GenreMoodSelector component
    ‚Üì
  user clicks genre ‚Üí handleGenreSelect()
    ‚Üì
  API call: GET /api/spotify/playlists/search?query={genre}
    ‚Üì
  PlaylistBrowser component
    ‚Üì
  user clicks playlist ‚Üí handlePlaylistSelect()
    ‚Üì
  route: /dashboard/swipe?mode=playlist&playlistId={id}
    ‚Üì
  useTrackQueue(mode='playlist', playlistId='{id}')
    ‚Üì
  API call: GET /api/spotify/playlists/{id}/tracks
    ‚Üì
  SwipePage displays playlist tracks
    ‚Üì
  user swipes, likes, saves (existing functionality)

  - Loading: Skeleton animation with 6 placeholder cards

DiscoverPage:
  - Two-step flow: Genre selection ‚Üí Playlist browsing
  - State: Tracks current step (genre | playlists)
  - Error: User-friendly error messages with recovery
  - Navigation: Back buttons at each step
  - API: Fetches from /api/spotify/playlists/search

useTrackQueue Hook:
  - Modes: 'recommendations' (default), 'playlist' (new)
  - Recommendations Mode:
    * Fetches: /api/spotify/search
    * Behavior: Auto-refill when queue low
    * Use: Infinite discovery with seed parameters
  - Playlist Mode:
    * Fetches: /api/spotify/playlists/{id}/tracks
    * Behavior: Load all tracks at once, no refill
    * Use: Browse specific playlist tracks
  - Exports: tracks, currentIndex, isLoading, error, playlistName, mode, etc.

SwipePage Enhancements:
  - Reads: mode and playlistId from query params
  - Header: Different for each mode

    * Playlist: Playlist name with back button
  - Progress: Mode-specific counter
    * Recommendations: "Showing X of Y tracks"
    * Playlist: "Track X of Y"
  - Navigation: Back to discovery button in playlist mode

API INTEGRATION REQUIREMENTS
================================================================================

Endpoint 1: GET /api/spotify/playlists/search
  Purpose: Search for playlists by genre/keyword
  Query Parameters:
    - query: string (genre name) - REQUIRED
    - limit: number (default 20, max 50) - OPTIONAL
    - offset: number (default 0) - OPTIONAL

  Expected Response:
    {
      "success": true,
      "data": {
        "playlists": [
          {
            "id": "spotify_playlist_id",
            "name": "Playlist Name",
            "description": "...",
            "imageUrl": "https://...",
            "trackCount": 50,
            "owner": "Spotify",
            "followerCount": 1000000
          }
        ],
        "total": 50,
        "limit": 20,
        "offset": 0
      }
    }

Endpoint 2: GET /api/spotify/playlists/:playlistId/tracks
  Purpose: Get all tracks from a specific playlist
  URL Parameters:
    - playlistId: string (Spotify playlist ID) - REQUIRED
  Query Parameters:
    - filterPreview: boolean (default true) - OPTIONAL

  Expected Response:
    {
      "success": true,
      "data": {
        "playlistId": "spotify_playlist_id",
        "playlistName": "Playlist Name",
        "tracks": [
          {
            "id": "spotify_track_id",
            "name": "Song Name",
            "artists": [
              { "id": "artist_id", "name": "Artist Name" }
            ],
            "album": {
              "id": "album_id",
              "name": "Album Name",
              "imageUrl": "https://..."
            },
            "durationMs": 210000,
            "previewUrl": "https://p.scdn.co/mp3-preview/...",
            "popularity": 75
          }
        ],
        "total": 50,
        "hasMore": false
      }
    }
